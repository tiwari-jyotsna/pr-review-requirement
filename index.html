Overview

The AI PR Review Agent is an automated system that reviews GitHub Pull Requests using Large Language Models (LLMs).

The agent analyzes code changes, applies engineering review rules, generates feedback, and posts review comments on the Pull Request.

The system is designed to run locally using Python scripts and supports multiple LLM providers.

Functional Requirements

  • Read GitHub Pull Requests using GitHub token authentication.
  • Fetch changed files and code diffs.
  • Send code changes to LLM models for analysis.
  • Apply review rules such as:
    • Coding standards validation
    • Architecture guideline enforcement
    • Performance checks
    • Security checks
  • Generate AI-based code review.
  • Classify issues by severity (Critical, Major, Minor).
  • Post inline comments on specific lines.
  • Post a summary review comment on the PR.
  • Support multiple LLM providers.
  • Run locally on a Windows machine using Python scripts.

Review Rules

During analysis, the PR Review Agent applies engineering review rules to detect potential problems and improvements.

Coding Standards

  • Variable naming conventions
  • Code readability
  • Method size and complexity
  • Proper error handling

Architecture Guidelines

  • Controllers should not contain business logic
  • Service layers should manage business rules
  • Avoid circular dependencies

Performance Checks

  • Avoid database queries inside loops
  • Detect inefficient data processing
  • Prevent unnecessary object creation

Security Checks

  • Detect SQL injection risks
  • Prevent hardcoded secrets
  • Ensure safe input validation

Processing Workflow


1. Provide GitHub Pull Request URL
2. Authenticate using GitHub token
3. Fetch PR files and diff
4. Build review context
5. Apply review rules
   - coding standards
   - architecture guidelines
   - performance checks
   - security checks
6. Send analysis request to LLM
7. Generate review insights
8. Create summary + inline comments
9. Publish review to GitHub PR

Core Components

Component Description
GitHub Adapter Handles authentication and API communication with GitHub
PR Data Collector Retrieves changed files, diffs, and metadata
Context Builder Builds structured prompt for LLM review
LLM Router Routes requests to selected LLM provider
Review Engine Applies rules and generates review feedback
Review Publisher Posts summary and inline comments to GitHub

Supported LLM Providers

  • OpenAI
  • Google Gemini
  • Anthropic Claude
  • AWS Bedrock Models

The architecture uses a provider abstraction layer allowing the system to switch between models without changing the core logic.
